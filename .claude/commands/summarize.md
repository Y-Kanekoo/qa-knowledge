# トピック横断の知見要約

指定トピックについて、収録済みエントリ群から横断的に知見を要約する。複数企業のアプローチを比較し、実践的な洞察を提供する。

## 引数

`$ARGUMENTS` = `<トピック>`

例: `テスト自動化`, `フレイキーテスト対策`, `QA組織設計`, `CI/CDパイプライン`, `E2Eテスト戦略`

## 実行手順

### ステップ1: 関連エントリ特定

entries/ 内の全 .md ファイルの frontmatter（qa_domains, tags）と本文を確認し、`$ARGUMENTS` のトピックに関連するエントリを特定する。

### ステップ2: 内容読み込み

関連エントリの全内容を Read で読み込む。

### ステップ3: 横断的知見サマリ生成

以下の構造でサマリを生成:

```markdown
## トピック: $ARGUMENTS

### 概観
（このトピックに関する収録エントリの概要。N社N件のエントリに基づく分析）

### 各社のアプローチ
（企業ごとのアプローチの要点を箇条書き）

### 共通点
（複数企業に共通する手法・考え方）

### 差異・特徴的なアプローチ
（企業間で異なる点、ユニークな手法）

### 定量データ（あれば）
（エントリ内に含まれる具体的な数値・成果）

### 実践への推奨事項
（このトピックに取り組む際のベストプラクティス。3-5点）

### 参照元エントリ
（分析に使用したエントリのリスト。ファイル名とタイトル）
```

### ステップ4: カバレッジ不足の指摘

このトピックについてエントリが少ない（3件未満）場合、`/search-web` での追加調査を提案する。
