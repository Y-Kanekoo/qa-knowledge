# エントリ間の比較分析

2つ以上のエントリを比較し、アプローチの違いや共通点を構造的に分析する。

## 引数

`$ARGUMENTS` = `<ファイル名1> <ファイル名2> [ファイル名3 ...]`

例: `google-test-sizes.md netflix-chaosmonkey.md`

ファイル名は entries/ ディレクトリ内のファイル名（パス不要）。部分一致でも検索する。

## 実行手順

### ステップ1: エントリ特定

`$ARGUMENTS` をスペースで分割し、各トークンに対応する entries/ 内のファイルを特定する。
- 完全一致を優先
- 見つからない場合はファイル名の部分一致で検索
- それでも見つからない場合はエラーを表示し、候補を提示

### ステップ2: エントリ読み込み

特定したエントリファイルの全内容（frontmatter + 本文）を Read で読み込む。

### ステップ3: メタデータ比較表

以下の形式で比較表を生成:

```
| 項目 | エントリA | エントリB |
|------|---------|---------|
| 企業 | Google | Netflix |
| 公開日 | 2024-03-15 | 2023-11-20 |
| 種別 | blog | conference_talk |
| QA領域 | test-strategy | reliability |
| 難易度 | advanced | intermediate |
| 言語 | en | en |
```

### ステップ4: 内容分析

以下の観点で比較分析を生成:

- **企業規模・業種の文脈**: 各社の背景がアプローチにどう影響しているか
- **QA手法の類似点**: 共通して採用している手法・考え方
- **QA手法の相違点**: アプローチが異なる部分とその理由の推察
- **適用条件の違い**: どのような組織・プロジェクトに適しているか
- **補完関係**: 両方を組み合わせることで得られる追加の知見

### ステップ5: 結論

比較から得られる主要な洞察を3点以内でまとめる。
