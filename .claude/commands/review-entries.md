# エントリ品質レビュー + 一括改善

低スコアエントリを検出し、概要・学び・関連エントリリンクを一括で改善する。

## 引数

`$ARGUMENTS` = `[--threshold N] [--fix]`

- `--threshold N`: このスコア以下のエントリを対象（デフォルト: 7）
- `--fix`: 検出した問題を自動修正する（省略時はレポートのみ）

## 実行手順

### ステップ1: 品質スコアリング

```bash
.venv/bin/python scripts/score_entries.py --threshold 7
```

### ステップ2: 問題分析

低スコアエントリごとに以下を確認:
- 概要セクションが50字未満 → 元記事を WebFetch で取得し概要を生成
- 「何が学べるか」が3点未満 → 元記事から学びを追加
- タグが3個未満 → 記事内容に基づきタグを追加
- industry / difficulty が未記入 → 推定して記入
- 関連エントリリンクがない → entries/ 内で同じ qa_domains / tags を持つエントリを検索しリンク

### ステップ3: 修正適用（--fix 指定時）

ユーザーに修正内容を一覧表示し、承認を得てから Edit ツールで修正する。

### ステップ4: バリデーション

```bash
.venv/bin/python scripts/validate_frontmatter.py
```

### ステップ5: レポート

修正前後のスコア比較テーブルを表示:

```
| ファイル | 修正前 | 修正後 | 改善点 |
|---------|--------|--------|--------|
| example.md | 5/10 | 9/10 | 概要追加、タグ追加、関連リンク追加 |
```
