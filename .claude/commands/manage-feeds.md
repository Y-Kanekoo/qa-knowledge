# RSSフィード管理

feeds.yml のフィード追加・一覧表示・動作確認を行う。

## 引数

`$ARGUMENTS` = サブコマンド

- `list` — 登録済みフィードの一覧表示
- `add <URL> --company <企業名> [--language <en|ja>] [--skip-filter]` — 新規フィード追加
- `test` — 全フィードの到達性・パース可能性を一括検証
- `check <フィード名>` — 特定フィードの最新エントリを表示

## 実行手順

### サブコマンド: list

feeds.yml を Read で読み込み、以下の形式で表示:

```
| # | 名前 | 企業 | URL | 言語 | フィルタ |
|---|------|------|-----|------|---------|
| 1 | Google Testing Blog | Google | https://... | en | スキップ |
| 2 | Netflix Tech Blog | Netflix | https://... | en | 有効 |
```

### サブコマンド: add

1. 指定URLにアクセスし、RSSフィードとしてパース可能か検証
2. パース成功時、フィードのタイトル・最新エントリ数件を表示して確認
3. ユーザーの承認を得てから feeds.yml に追加（Edit ツール使用）
4. 追加後のフィード数を表示

feeds.yml のフォーマット:
```yaml
  - name: <フィード名>
    company: <企業名>
    url: <URL>
    language: <en|ja>
    skip_keyword_filter: true  # --skip-filter 指定時のみ
```

### サブコマンド: test

1. feeds.yml の全フィードURLにアクセス
2. 各フィードの到達性（HTTP ステータス）とパース可能性を確認
3. 結果を以下の形式で表示:

```
| フィード | 到達性 | パース | 最新記事日 |
|---------|--------|-------|----------|
| Google Testing Blog | OK | OK | 2026-03-15 |
| 壊れたフィード | NG (404) | - | - |
```

### サブコマンド: check

1. 指定フィード名で feeds.yml を検索（部分一致）
2. フィードURLを取得してパース
3. 最新5件のエントリを表示:

```
| # | タイトル | 公開日 | URL |
|---|---------|--------|-----|
| 1 | ... | 2026-03-15 | https://... |
```
