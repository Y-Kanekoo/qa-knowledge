# 自然言語でエントリ検索

自然言語のクエリでエントリを検索し、条件に合致するエントリを一覧表示する。

## 引数

`$ARGUMENTS` = `<検索クエリ>`

例:
- `Googleのテスト自動化に関する記事`
- `ci-cd ドメインの日本語記事`
- `2025年以降に追加されたadvanced難易度の記事`
- `フレイキーテスト`
- `conference_talk タイプのエントリ`

## 実行手順

### ステップ1: クエリ解析

`$ARGUMENTS` を解析し、以下のフィルタ条件を抽出:

- **company**: 企業名（部分一致）
- **qa_domains**: QAドメイン（完全一致）
- **tags**: タグ（部分一致）
- **content_type**: コンテンツ種別（完全一致）
- **language**: 言語（en / ja）
- **difficulty**: 難易度（beginner / intermediate / advanced）
- **日付範囲**: published_at または added_at の範囲
- **フリーテキスト**: 上記に該当しないキーワードは概要・本文で検索

### ステップ2: エントリ読み込み

entries/ 内の全 .md ファイル（_template.md 除く）を Glob で検索し、各ファイルの frontmatter と本文を Read で読み込む。

### ステップ3: フィルタリング

ステップ1で抽出した条件でフィルタリングする。複数条件はAND結合。

### ステップ4: 結果表示

マッチしたエントリを以下の形式で表示:

```
**N件のエントリが見つかりました**

| タイトル | 企業 | QA領域 | 公開日 | ファイル |
|---------|------|--------|--------|---------|
| ... | Google | test-automation | 2024-03-15 | google-test-sizes.md |
```

0件の場合は「条件に合致するエントリはありません」と表示し、条件の緩和を提案する。
