# RSS巡回で新着QA記事を発見

feeds.yml に定義されたRSSフィードを巡回し、未収録の新着QA記事を一覧表示する。

## 引数

`$ARGUMENTS` = `[--days N] [--add]`

- `--days N`: 過去N日以内の記事に限定（デフォルト: 30）
- `--add`: 発見した記事の中から選択してエントリ追加モードに入る

## 実行手順

### ステップ1: RSS巡回

```bash
.venv/bin/python scripts/check_rss.py
```

出力から新着記事のリストを取得する。

### ステップ2: 結果表示

新着記事を以下の形式で一覧表示:

```
| # | フィード | タイトル | 公開日 | URL |
|---|---------|---------|--------|-----|
| 1 | Google Testing Blog | ... | 2026-03-15 | https://... |
| 2 | Netflix Tech Blog | ... | 2026-03-12 | https://... |
```

新着がない場合は「新着QA記事はありません」と表示。

### ステップ3: 追加モード（--add 指定時）

新着記事がある場合:
1. ユーザーに追加したい記事の番号を選択させる
2. 選択された記事について、/add-entry と同等の処理を実行:
   - scaffold.py で骨格生成
   - WebFetch で記事取得
   - AI がメタデータ・概要・学びを自動補完
   - validate_frontmatter.py で検証
3. 複数選択された場合は順番に処理する
