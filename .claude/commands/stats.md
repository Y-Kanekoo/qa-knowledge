# ナレッジベース統計・カバレッジ分析

entries/ 内の全エントリを分析し、統計情報・QAドメインカバレッジ・傾向を可視化する。

## 引数

`$ARGUMENTS` = `[--detail]`

- `--detail`: 詳細モード（企業別エントリ数、最近30日の追加、deprecated一覧を追加）

## 実行手順

### ステップ1: データ収集

entries/ 内の全 .md ファイル（_template.md を除く）を Glob で検索し、各ファイルの frontmatter を Read で読み込む。

### ステップ2: 基本統計

以下を集計してテーブル表示:

- **総エントリ数**
- **企業数**（ユニークな company 値）
- **言語別内訳**（en / ja）
- **content_type 別分布**（9種の内訳）

### ステップ3: QAドメインカバレッジ

12領域すべてについてエントリ数を集計し、カバレッジ率を表示:

```
| QAドメイン | エントリ数 | カバレッジ |
|-----------|----------|----------|
| test-strategy | 5 | ████████░░ |
| test-automation | 8 | █████████░ |
| ai-testing | 0 | ░░░░░░░░░░ |  ← 未カバー
...
```

未カバー（0件）の領域を特に強調する。

### ステップ4: タグ頻度ランキング

タグの出現頻度上位20を表示。

### ステップ5: 月別追加推移

added_at の月別にエントリ数を集計し、推移を表示。

### ステップ6: 推奨フィールド記入率

industry と difficulty の記入率を計算して表示。記入率が50%未満の場合は警告。

### ステップ7（--detail 時のみ）: 詳細情報

- **企業別エントリ数**: 多い順にテーブル表示
- **最近30日に追加されたエントリ**: タイトル・企業・追加日の一覧
- **deprecated エントリ**: deprecated: true のエントリ一覧（あれば）

### ステップ8: 改善提案

分析結果に基づき、具体的な改善提案を最大3つ提示:
- 未カバーのQAドメインへの記事追加提案
- 記入率が低い推奨フィールドの補完提案
- content_type の偏りがあれば多様化の提案
